# Labs 页面系统说明文档

> 本文档用于说明 **Labs 页面系统**
> 的整体设计理念、功能结构、交互逻辑、匹配规则与扩展规范。\
> 适用于维护、扩展或二次开发本系统。

------------------------------------------------------------------------

## 1. 系统定位

Labs 是整个博客体系中的 **实验与交互展示中心**，不同于 Notes（笔记）：

-   Notes：偏 **知识沉淀 / 线性阅读**
-   Labs：偏 **交互实验 / 可视化 / 工具化体验**

核心目标：

> **让访问者"一眼能分清楚 + 点进去就能玩 / 用 / 看懂"。**

------------------------------------------------------------------------

## 2. 整体架构

### 2.1 页面结构

    Labs Page
    ├─ Header（页面标题与说明）
    ├─ Sidebar（左侧导航栏）
    │  ├─ 预设分类（Segmented Control）
    │  ├─ Tags（默认折叠，可展开）
    │  └─ 折叠/展开控制
    └─ Main Content
       ├─ 搜索栏
       ├─ 实验卡片网格
       └─ 空状态提示

### 2.2 数据来源

Labs 数据来自本地静态注册表：

    /labs/_registry.ts

每个 Lab 至少包含：

``` ts
{
  slug: string
  title: string
  description: string
  date: string
  tags: string[]
}
```

------------------------------------------------------------------------

## 3. 分类系统设计（Preset Categories）

### 3.1 主分类一览

Labs 采用 **体验导向型分类**，共 4 类：

  分类         代号   用户心智
  ------------ ------ ------------------
  小游戏       game   "我想玩"
  实验仿真     sim    "如果这样会怎样"
  原理可视化   viz    "我想看懂"
  小工具       tool   "我想用"

------------------------------------------------------------------------

### 3.2 主分类匹配词池说明（非常重要）

> **分类不是手写的，而是由 tags + 匹配词池自动推断。**

#### 3.2.1 小游戏（game）

**设计语义**：\
可玩性、探索性、结果非唯一。

**匹配词池：**

    game, 游戏, 小游戏, play, fun, toy,
    puzzle, 解谜, 闯关, physics, 物理

**典型 Lab 示例：** - 物理小游戏 - Emoji 合成 - 交互式解谜

------------------------------------------------------------------------

#### 3.2.2 实验仿真（sim）

**设计语义**：\
系统状态、规则推演、过程模拟。

**匹配词池：**

    simulation, sim, 仿真, 模拟,
    experiment, 实验,
    system, 系统,
    model, 模型,
    state-machine, 状态机,
    process, 流程

**典型 Lab 示例：** - 聊天模拟器 - 系统行为演示 - 状态流转实验

------------------------------------------------------------------------

#### 3.2.3 原理可视化（viz）

**设计语义**：\
抽象概念 → 视觉理解。

**匹配词池：**

    visualization, viz, 可视化,
    interactive, 交互,
    algorithm, 算法,
    math, 数学,
    dsp, 信号, signal,
    graph, 图,
    pathfinding, 寻路,
    fourier, 傅里叶,
    principle, 原理

**典型 Lab 示例：** - 傅里叶变换可视化 - 寻路算法动画 - 数学变换交互

------------------------------------------------------------------------

#### 3.2.4 小工具（tool）

**设计语义**：\
功能明确、可重复使用。

**匹配词池：**

    tool, tools, 工具, 小工具,
    utility, utils,
    generator, 生成器,
    converter, 转换器,
    formatter, 格式化,
    editor, 编辑器,
    assistant, 助手,
    encode, decode, 编码

**典型 Lab 示例：** - 文本生成器 - 编码/解码工具 - 配置编辑器

------------------------------------------------------------------------

### 3.3 主分类优先级规则

一个 Lab 可能同时命中多个分类，因此采用固定优先级：

    tool > game > sim > viz

目的： - 保证 **唯一主分类** - 让"用途明确的内容"优先

------------------------------------------------------------------------

## 4. 视觉系统（主分类色系）

> **整个 Labs 页面只存在 4 套主色系**

  分类   渐变色
  ------ ----------------
  game   Rose → Orange
  sim    Emerald → Teal
  viz    Sky → Indigo
  tool   Amber → Yellow

这些颜色用于： - 卡片预览区背景 - hover 阴影 - 标题 hover 色 - CTA
文案 - 主分类徽章

### 5.3 Icon 兜底规则（非常关键）

> **如果没有任何主题命中：**

-   预览区 icon **必须使用主分类 icon**

    -   永远有 icon
    -   视觉语义不崩坏

## 6. 左侧导航栏交互

-   支持折叠为窄圆角胶囊条
-   展开按钮 sticky 跟随滚动
-   折叠后主内容区变宽
-   Preset 分类使用 iOS 风格 segmented control
-   动效仅使用 `transform`（高性能）

------------------------------------------------------------------------

## 7. Tags 系统

-   默认折叠
-   展开后约 6 行高度
-   内部滚动
-   Tags 仅作为探索辅助，不参与主分类判定

------------------------------------------------------------------------

## 8. 搜索与性能原则

-   搜索字段：
    -   title
    -   description
    -   tags
-   前端过滤
-   `useMemo` 缓存
-   无额外网络请求

------------------------------------------------------------------------

## 9. 新增 Lab 的推荐实践

``` ts
{
  title: "傅里叶变换交互实验",
  description: "通过交互方式理解傅里叶变换",
  tags: ["DSP", "傅里叶", "可视化"]
}
```

**原则：** - tags 写"语义事实" - 系统负责分类与视觉

------------------------------------------------------------------------

## 10. 禁止事项（设计红线）

❌ 手写分类字段\
❌ 给每个 tag 设计 icon\
❌ 添加第五套主色系\
❌ 让视觉规则依赖具体 Lab

------------------------------------------------------------------------

## 11. 设计哲学总结

> **一致的是语义，而不是外观。**\
> **系统负责美感，作者只负责内容。**
